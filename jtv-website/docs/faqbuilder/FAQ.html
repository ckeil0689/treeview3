1. Java TreeView can't see the cdt file I downloaded! It's in the directory, but doesn't show up when I go to the file open dialog.

Java TreeView will only open files which have a ".cdt" extension. Make sure that your file ends in ".cdt". Note that java treeview can parse ".pcl" files as well, but you must rename them. I will probably change the filename filtering in a later release so that you don't need to rename pcls. Check the feature page when I get around to making it for more feature related things.



2. When I load a new file or click on a new cluster or something, there is some weird drawing artifact, such as old text still being displayed. How can I get rid of this?

The workaround is to just resize the component with the weirdness. I will actually fix the bug at some point.



3. MAC ONLY: Why does my freshly installed Java Treeview crash when I open a file?

Try rebooting. I have heard MRJ can be unhappy unless you reboot after the install.



4. I have human/mouse/lymphochip genes. How can I link to SOURCE?

There should be a preset which is called Source_SUID. Under the document menu, select Gene settings and then click on the appropriate preset. If the preset is mssing, try entering the following custom URL: From Max Diehn,
Select Document:Url Options... and enter the following custom format: 
 http://genome-www4.stanford.edu/cgi-bin/SMD/source/sourceResult?option=CloneID&choice=Gene&criteria=HEADER
This will work if you extract your CDT with clone ID as the unique identifier (which I think is what most people do). If you used LUID as the identifier, then enter the following custom format: 
 http://genome-www4.stanford.edu/cgi-bin/SMD/source/sourceRedirectLUID.pl?luid=HEADER



5. How can I select the parent of a node? Up arrow does something counterintuitive.

Java TreeView has context-sensitive keyboard event processing. What this means is that the particular tree panel must have the focus in order for it to get the keyboard event. This is necessary, since the up arrow selects the parent node in both the array and gene trees. If it weren't focus-sensative, it wouldn't know which tree to select the parent of. Unless only one tree had a node selected, I suppose. You get the idea. Anyways, moving the mouse over the panel will give it the focus. The panel with the focus should have a heavier black border.

8. I typed something into a text field and closed the dialog window, but the program ignored me. What gives?

You have stumbled into an issue with how java deals with string input. You basically need to press the enter key after you paste in the url in order for the program to actually register it. This is not particularly elegant... I initially wrote java treeview with a buggy but fast toolkit called AWT. I'm slowly replacing these components with SWING components, which are slightly slower but much more stable. In the meantime, remember to push enter after you change a value in a textfield- this applies to all text fields, including the contrast settings, if you change them directly.
