SECTION 1: BITBUCKET PULL REQUEST REVIEW
++++++++++++++++++++++++++++++++++++++++

USAGE:
1. The CCB Librarian reviews a submitted pull request via BitBucket.[2]
2. When the level of effort of the change is Medium+, the Librarian schedules a
   document review meeting with the CM Leader.
3. The CCB Librarian sends a link to the pull request, along with a suggested
   meeting time, to a minimum of 2 reviewers of his/her choice, including the CM
   Leader.
4. Each reviewer inspects the changes via the diff panes on the BitBucket pull
   request page.  Issues/problems are commented on by hovering over the
   associated line number.  For each issue comment the below review template is
   pasted and filled out.  Issues without a clearly applicable line number may
   be entered using the comment button at the top-right of the diff pane.
5. The reviewer lastly, adds a comment to the pull request with a recommendation
   as to whether they feel a meeting is necessary.  The recommendation can be
   one of: Meeting/Accept As-Is/Accept after issues addressed.
6. If a period of 2 weeks passes without a scheduled meeting or a response from
   the CCB Librarian, the merge may proceed as-is.  A merge request may be
   resubmitted with changes if initially rejected.

SECTION 2: MANUAL MERGE/COMMIT REQUEST REVIEW
+++++++++++++++++++++++++++++++++++++++++++++

USAGE:
1. A merge request form is submitted for review to the CCB Librarian including a
   branch or a diff.[2]
2. When the level of effort of the change is Medium+, the Librarian schedules a
   document review meeting with the CM Leader.
3. The CCB Librarian (adds line numbers and) distributes the document (with
   "_inspection" appended to the file name before the file extension), along
   with a suggested meeting time, to a minimum of 2 reviewers of his/her choice,
   including the CM Leader.
4. Each reviewer inspects the changes, notes issues using the below review
   template (including the header), and makes a recommendation as to whether a
   meeting is necessary.  The reviewer appends their first name to the file name
   before the file extension.
5. Completed forms are submitted to the CM Leader 24 hours before the meeting.
